# Obsidian Workflow

Use this skill when project decisions, implementation plans, or acceptance records should be persisted into an Obsidian vault.

## Instructions

- Write durable project records into the existing vault structure instead of the source repository when the repository is public or should not contain design notes.
- Use an exact vault-relative path.
- Avoid duplicate H1 titles when the filename already provides the title.
- After writing, read the note back and verify the important sections are present.
- Record concrete evidence: branch, commit, commands run, and verification results.

## Output Shape

- Current state
- Decisions
- Implementation or acceptance scope
- Verification evidence
- Follow-up actions
